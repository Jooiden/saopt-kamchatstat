# САОПТ Камчатстат (Система автоматизированной обработки психологических тестов)

**САОПТ Камчатстат** — специализированный программный комплекс (автоматизированное рабочее место психолога), разработанный для Территориального органа Федеральной службы государственной статистики по Камчатскому краю (Камчатстат). Система предназначена для проведения психологической диагностики сотрудников, автоматического расчета показателей по методикам и централизованного электронного хранения результатов.

Проект разработан в рамках выпускной квалификационной работы (дипломного проекта).

---

### 🚀 Основные возможности

* **Автоматизация тестирования:** Интерактивное прохождение встроенных психологических методик (Опросник Басса-Перри, Методика суицидального риска Т.Н. Разуваевой, Диагностический опросник СПК группы).
* **Математическое ядро:** Мгновенный расчет первичных («сырых») баллов и автоматическая нормализация результатов — перевод в стандартизированную 9-балльную шкалу стенайнов на основе нормативных коэффициентов ($\mu$ и $\sigma$) из базы данных.
* **Многопользовательский режим:** Разграничение прав доступа для трех ролей: *Администратор*, *Психолог*, *Сотрудник*.
* **Автоматизация отчетности:** Прямая генерация индивидуальных заключений в формате `.docx` (MS Word) и сводных групповых ведомостей в формате `.xlsx` (MS Excel) без жесткой зависимости от установленного пакета MS Office.
* **Надежное локальное хранение:** Использование встраиваемой СУБД SQLite обеспечивает полную конфиденциальность и локальную обработку персональных данных в соответствии с ФЗ №152 «О персональных данных».

### 💻 Технологический стек

* **Графический интерфейс (GUI):** CustomTkinter (современный адаптивный дизайн с поддержкой темной/светлой темы)
* **База данных:** SQLite 3 (модуль `sqlite3`)
* **Экспорт данных:** `python-docx`, `openpyxl`

### 🔒 Безопасность и защита данных

Программный комплекс разработан с учетом требований безопасности для автоматизированных рабочих мест государственных учреждений:
1. **Локальный контур:** Все данные хранятся и обрабатываются строго на целевом компьютере без отправки во внешние облачные сервисы, что исключает утечку информации.
2. **Разграничение прав (RBAC):** Каждая роль имеет строго ограниченный набор GUI-компонентов и SQL-привилегий (например, сотрудник имеет доступ только к форме тестирования и не видит архив результатов).
3. **Безопасность авторизации:** Реализованы модули верификации учетных записей, а также архитектурная возможность интеграции механизмов двухфакторной аутентификации (2FA) для защиты личных кабинетов администраторов и психологов.

### 🗄️ Структура базы данных (`kamstat.db`)

Реляционная база данных состоит из следующих ключевых сущностей:
* `users` — учетные записи пользователей, их роли (хешированные пароли, логины, ФИО).
* `tests` — справочник психологических методик и шкал.
* `questions` — перечень вопросов, вариантов ответов и соответствующих им весовых коэффициентов («ключей» теста).
* `results` — архив записей: сырые баллы, итоговые стенайны, дата прохождения и ID сотрудника.

---

### 📂 Структура проекта и назначение модулей

* `main.py` — Точка входа в приложение. Инициализация компонентов и запуск главного UI-цикла.
* `main_menu.py` — Логика главного меню системы и навигация между разделами.
* `auth.py` — Модуль авторизации, верификация учетных записей и разграничение прав доступа.
* `gui.py` — Описание графических элементов интерфейса (формы, фреймы, кнопки).
* `Processor.py` — Вычислительное ядро. Алгоритмы расчета сумм баллов и конвертации результатов в стенайны.
* `data_io.py` — Модуль ввода-вывода, выполнение SQL-запросов к локальной СУБД.
* `charts.py` — Визуализация данных, построение графиков и многофакторных психологических профилей.
* `reports.py` — Модуль автоматической генерации и экспорта документов Word/Excel по шаблонам.
* `kamstat.db` — Файл реляционной базы данных SQLite (справочники, тесты, вопросы, пользователи, архив результатов).
* `system_log.txt` — Журнал системных событий для логирования действий пользователей и фиксации ошибок.
* `requirements.txt` — Файл внешних программных зависимостей (библиотек).

### 🛠️ Требования и развертывание

#### Системные требования:
* **ОС:** Windows 7/10/11
* **ЦП:** Архитектура x64, от 1.6 ГГц
* **ОЗУ:** не менее 2 Гб (рекомендуется 4 Гб)
* **Свободное место:** не менее 200 Мб

#### Инструкция по запуску:
1. Скопируйте все файлы программного пакета в рабочую директорию на жестком диске.
2. Установите необходимые зависимости (при запуске из исходного кода):
   ```bash
	pip install -r requirements.txt
3. Запустите систему через консоль или терминал среды разработки:
	```bash
	python main.py

⚠️ Важное примечание: Не перемещайте и не переименовывайте файл базы данных kamstat.db относительно исполняемых модулей приложения, иначе нарушится интеграция данных и программа выдаст ошибку инициализации СУБД.
